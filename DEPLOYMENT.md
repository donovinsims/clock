# Vercel Deployment Guide

This project is a **TanStack Start** application with **Nitro** server framework, configured for Vercel deployment.

## ✅ Fixed Issues

The original Lovable.dev setup didn't work with Vercel because:
1. Missing Nitro configuration for Vercel preset
2. Incorrect build output structure
3. The app was generating output in `dist/` instead of `.vercel/output/`

## 🔧 Configuration

### Files Created/Modified

1. **`nitro.config.ts`** - Configures Nitro to use Vercel preset
2. **`vercel.json`** - Simplified to let Nitro handle the build configuration
3. Removed old `dist/` directory

### Build Output

- Build command: `npm run build`
- Output directory: `.vercel/output/` (auto-generated by Nitro)
- Static assets: `.vercel/output/static/`
- Server function: `.vercel/output/functions/__server.func/`

## 🚀 Deployment Commands

### Deploy to Production
```bash
npm run build
vercel deploy --prod --prebuilt
```

### Deploy to Preview
```bash
npm run build
vercel deploy --prebuilt
```

### Build Only (for testing)
```bash
npm run build
```

## 📝 Important Notes

1. **Always build before deploying** - Vercel needs the `.vercel/output/` directory
2. **Use `--prebuilt` flag** - This tells Vercel to use the pre-built output
3. **Nitro handles everything** - Don't manually configure `vercel.json` build settings
4. **The `.vercel/output/` directory is gitignored** - It's generated on each build

## 🌐 Live URLs

- Production: https://clock-mu-henna.vercel.app
- Inspect: Check Vercel dashboard for deployment details

## 🔍 Troubleshooting

If you get a 404 error:
1. Make sure `nitro.config.ts` exists with the Vercel preset
2. Run `npm run build` to generate `.vercel/output/`
3. Check that `.vercel/output/functions/__server.func/` exists
4. Deploy with `vercel deploy --prod --prebuilt`

If the build fails:
1. Check Node.js version (requires >= 20)
2. Clear caches: `rm -rf node_modules .nitro .tanstack`
3. Reinstall: `npm install`
4. Rebuild: `npm run build`
